## 0. Regola di utilizzo

Questo file NON è una cartella immagini completa.

È una lista curata di reference pubbliche da raccogliere manualmente, in modo piccolo, pulito e utile per brandizzare Walbox V2 in stile Walrus.

Usare così:

```text
1. Apri i link sorgente.
2. Salva solo le immagini indicate.
3. Rinominale con i file name suggeriti.
4. Inseriscile nella cartella /references/walrus-brand/.
5. Usa le immagini come ispirazione visiva e contesto per Antigravity.
```

Non usare così:

```text
- scraping massivo;
- download automatico da social;
- uso commerciale diretto senza autorizzazione;
- modifica del core Walbox;
- trasformazione di Walbox in gestionale;
- training o dataset generico;
- raccolta casuale di 100 foto.
```

Nota legale/pratica:

```text
Le immagini sono pubbliche ma restano di proprietà dei rispettivi autori/piattaforme/locali.
Per demo interna e reference creativa va bene usarle come materiale di studio.
Per pubblicazione, sito, ads, pitch pubblico o materiale commerciale serve validare con il locale o usare asset autorizzati.
```

## 1. Obiettivo del reference pack

Creare un set visivo piccolo, leggibile e operativo per aiutare gli agenti a capire:

- logo e insegna;
- atmosfera del locale;
- materiali reali;
- palette reale;
- luci;
- tono social;
- dettaglio pub/birre/drink;
- elementi iconici da trasformare in UI Walbox;
- cosa evitare per non rendere Walbox troppo generica.

Il pack deve aiutare soprattutto:

```text
Creative Director Agent → direzione visiva, moodboard, UI vibe
Venue Experience Strategist → format serata, momenti, contenuti social
Frontend Agent → palette, texture, layout, asset inspiration
Antigravity → brandizzazione concreta delle schermate Walbox V2
```

## 5. Immagini opzionali

Raccogli solo se servono dopo il primo giro:

1. Altri post Instagram recenti con pubblico/serata, se mostrano atmosfera e non solo persone in posa.
2. Reel pubblici con luci/locale/staff, solo come screenshot reference.
3. Foto del menù food se vuoi creare una demo futura “Tourist Mode / Menu multilingua”.
4. Dettagli pareti/quadri, solo se visivamente forti.
5. Foto birre specifiche, solo se diventano mood/badge o gamification fake.
6. Foto dello staff, solo se autorizzate e utili a tono/community.

---

## 6. Immagini da evitare

Evitare:

```text
- foto con volti di clienti troppo riconoscibili se non servono;
- immagini sfocate o low quality;
- foto solo food generiche senza brand Walrus;
- duplicati della stessa insegna;
- immagini di altre sedi confuse con Milano, salvo nota esplicita;
- immagini di altri locali simili;
- screenshot social pieni di commenti/dati personali;
- foto troppo vecchie se non rappresentano più il locale;
- immagini usate solo perché “belle” ma non utili alla UI;
- elementi che spingono Walbox verso gestionale, prenotazioni, conti aperti reali o loyalty backend ora.
```

Regola:

```text
Se un’immagine non aiuta UI, tono, atmosfera o format serata, non entra nel pack.
```

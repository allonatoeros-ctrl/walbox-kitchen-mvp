# Walrus Visual Polish

## Mission

Improve visual quality in Walrus/Walbox screens while preserving product logic.

## Brand direction

Walrus/Walbox should feel:

```text
pub
ironic
bold
poster-like
social
warm
not corporate
not generic SaaS
not sterile
```

## Visual vocabulary

Use when appropriate:

```text
dark brown / black backgrounds
cream typography
yellow/orange CTA accents
bold condensed poster-style hierarchy
food-poster energy for Kitchen
mobile-first composition
clear spacing
strong but controlled microinteractions
```

## Do not

- invent features
- change routes
- change data/state logic
- add backend/API
- add dependencies
- touch Supabase/Spotify legacy core
- overuse “walrus/tricheco” copy
- use generic corporate SaaS tone

## Use when

- polishing a screen after logic is stable
- adapting Figma reference into existing UI
- improving perceived value
- making a screen client-grade

## Output

1. Visual goal
2. Files touched
3. Brand decisions
4. What was not touched
5. Mobile QA notes
6. Next single action
